# Image Dimensions Expectations

Based on the workflow scripts in `scripts/`, here are the expectations for image dimensions across the different datasets:

## 1. Images Dataset (Raw full-size scans)
These are the raw images placed in the folder structure created by `1-setup_folder_structure.py` and processed by `2-sample_data.py` and `4-run_inference.py`.

* **Minimum Dimensions**: Images must be **at least as large as the `patch-size` / `input-size`** in both height and width (default is 128x128 pixels).
* **Why**: 
  * `2-sample_data.py` uses a sliding window to extract patches. It will explicitly skip any image where either dimension is smaller than the requested `patch-size`.
  * `4-run_inference.py` uses `DLTracer`, which requires images to be $\ge$ `input-size` so it can either process them directly or split them into appropriately sized tiles for the neural network.
* **Shape**: Typically large 2D (H, W) or 3D (H, W, C) `.tif` files.

## 2. Tracings Datasets (Training and Test sets)
These are the sampled patches generated by `2-sample_data.py` and used by `3-train_model.py`. This applies to `img.tif`, `outer_mask.tif`, and the manually annotated `tracings.tif`.

* **Exact Dimensions**: Images must be **exactly square patches matching the `patch-size`** (default 128x128 pixels).
* **Why**:
  * `2-sample_data.py` generates these patches using a fixed window shape of `(patch_size, patch_size)`.
  * `3-train_model.py` feeds these directly into the UNet model, which expects inputs of size `input-size` $\times$ `input-size`. The dimensions of `tracings.tif` must perfectly match `img.tif` and `outer_mask.tif` for the loss functions and metrics to compute correctly.
